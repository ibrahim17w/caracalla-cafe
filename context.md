# Caracalla Cafe - Project Context

## Overview
نظام طلبات وقائمة لمقهى كاراكالا في سوريا. الزبون يمسح رمز QR ليفتح القائمة، يختار الأصناف مع إضافات، ويختار بين الأكل داخل المقهى أو التوصيل. المالك/الكاشير يرى الطلبات ويحدث حالتها. السائق يرى طلبات التوصيل وموقعها على الخريطة.

## File Map

### Backend

| File | Purpose |
|------|---------|
| `server.js` | خادم Express على المنفذ 3015. يحتوي على جميع واجهات API: المصادقة (مالك/سائق)، الإعدادات، الأقسام، الأصناف (مع رفع صور)، الطلبات، رمز QR، الإحصائيات. حماية من تكرار محاولات تسجيل الدخول. |
| `package.json` | اعتماديات Node.js: express, pg, cors, qrcode, body-parser, dotenv, multer, jsonwebtoken. |
| `.env` | ملف الإعدادات. عدل كلمة مرور PostgreSQL وكلمات مرور المالك والسائق والمنفذ هنا. |
| `init.sql` | أنشئ قاعدة البيانات `caracalla_cafe` في pgAdmin ثم شغّل هذا الملف. ينشئ الجداول فقط بدون بيانات تجريبية. |

### Frontend (الزبون)

| File | Purpose |
|------|---------|
| `public/menu.html` | صفحة القائمة للزبائن. تصميم RTL عربي. تبويبات الأقسام، بطاقات الأصناف مع صور، نافذة تخصيص الإضافات، سلة الطلبات. |
| `public/app.js` | منطق صفحة الزبون. تحميل القائمة، إدارة السلة، إرسال الطلب مع الموقع الجغرافي، العملة بالليرة السورية (ل.س). زر "موقع المقهى" يفتح خريطة مع موقع المقهى وموقع الزبون. زر "حالة طلبي" لتتبع الطلب وتأكيد الاستلام. |

### Frontend (المالك/الكاشير - صفحة واحدة)

| File | Purpose |
|------|---------|
| `public/owner.html` | لوحة تحكم موحدة للمالك والكاشير. شاشة تسجيل دخول محمية بكلمة مرور. إنشاء رمز QR (مشاركة/طباعة)، إحصائيات، إدارة الأصناف (إضافة/تعديل/حذف مع رفع صور)، إدارة الأقسام (سحب وإفلات لإعادة الترتيب)، عرض جميع الطلبات مع خريطة موقع التوصيل، إعدادات المقهى (اسم، هاتف، عنوان، منطقة زمنية، شعار، موقع على الخريطة)، إعدادات الفاتورة (حقول مخصصة ديناميكية، نص أسفل الفاتورة). |
| `public/owner.js` | منطق لوحة المالك. مصادقة JWT، رفع الصور عبر FormData، إدارة الأقسام والأصناف، عرض الطلبات مع الخرائط، إنشاء فواتير قابلة للطباعة، إدارة الإعدادات. |

### Frontend (السائق)

## Database Schema

- **settings** - إعدادات المقهى (اسم، هاتف، عنوان، منطقة زمنية، شعار، موقع، إعدادات الفاتورة)
- **categories** - أقسام القائمة مع ترتيب (sort_order)
- **items** - أصناف القائمة مع السعر والوصف والصورة والتوفر والمخزون
- **item_additions** - إضافات مرتبطة بكل صنف
- **orders** - الطلبات: اسم الزبون (اختياري)، هاتف، رقم الطاولة، نوع الطلب (داخل/توصيل)، العنوان، الإحداثيات، الحالة، المبلغ الإجمالي
- **order_items** - بنود الطلب
- **order_item_additions** - الإضافات المختارة لكل بند

## Order Status Flow

- `pending` → `preparing` → `ready` → `delivering` → `completed`
- `cancelled` (من أي حالة قبل completed)
- السائق يرى الطلبات فقط عندما تكون `ready` أو `delivering`
- الزبون يستطيع تتبع الطلب عبر رقم الطلب
- الزبون يؤكد استلام الطلب (يحول الحالة إلى `completed`)

## Security

- الزبون لا يمكنه الوصول لصفحات المالك أو السائق (مصادقة JWT مطلوبة)
- 5 محاولات تسجيل دخول فاشلة = حظر IP لمدة 15 دقيقة
- كلمات المرور منفصلة: `OWNER_PASSWORD` و `DRIVER_PASSWORD` في ملف `.env`
- رمز JWT صالح لـ 8 ساعات
- السائق لا يمكنه الوصول لصفحة المالك (يحتاج كلمة مرور المالك)
- المالك يمكنه الوصول لصفحة السائق بدون إعادة تسجيل الدخول (نفس التوكن)

## URLs (after running `node server.js`)

- قائمة الزبون: `http://localhost:3015/menu.html`
- لوحة المالك/الكاشير: `http://localhost:3015/owner.html`
- بوابة السائق: `http://localhost:3015/driver.html`

## Setup Steps

1. أنشئ قاعدة بيانات `caracalla_cafe` في pgAdmin
2. شغّل `init.sql` في Query Tool
3. عدل `.env`:
   - `DB_PASSWORD=your_password_here` → كلمة مرور PostgreSQL
   - `OWNER_PASSWORD=owner123` → كلمة مرور المالك
   - `DRIVER_PASSWORD=driver123` → كلمة مرور السائق
4. `npm install` ثم `node server.js`

## To Edit / Add Features

- **تغيير كلمة المرور أو المنفذ**: عدل ملف `.env`
- **تغيير التصميم**: `public/style.css`
- **تغيير قائمة الزبون**: `public/menu.html` + `public/app.js`
- **تغيير لوحة المالك**: `public/owner.html` + `public/owner.js`
- **تغيير السائق**: `public/driver.html` + `public/driver.js`
- **إضافة API جديد**: `server.js`
- **تغيير هيكل قاعدة البيانات**: عدل `init.sql` (شغّله في pgAdmin)، ثم عدل الاستعلامات في `server.js`
